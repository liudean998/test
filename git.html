版本控制系统

1.本地版本控制系统

使用软件记录文件的不同版本，提高了工作效率。 缺点：单机运行，不支持多人协作开发。版本数据库故障后所有历史更新记录会丢失。

2.集中化版本控制系统

基于服务器客户端的运行模式，服务器保存文件的所有更新记录，客户端只保留最新的文件版本。 缺点：不支持离线提交版本更新。中心服务器崩溃后，所有人无法正常工作。版本数据库故障后，所有历史记录丢失。典型代表SVN

3.分布式版本控制系统

基于服务器客户端的运行模式，服务器保存文件的所有更新版本，客户端存储服务器的完整备份，并不只保留文件的最新版本。 联网运行支持多人写作开发，断网后支持离线本地提交，服务器损坏后可使用任何一个客户端的备份进行恢复。典型代表git

Git

特点：断网后依旧可以再本地对项目进行版本管理，联网后把本地修改的记录同步到云端服务器即可
优点：版本切换时非常快，因为每个版本都是完整的文件快照（备份），切换版本时直接恢复目标版本的快照即可
缺点：占用磁盘空间较大

Git的三个区域

1.工作区2.暂存区3.Git仓库

Git的三个状态

已修改 modified 已暂存 staged 已提交 committed

获取git仓库的两种方式

  • 将尚未进行版本控制的本地目录转换为git仓库： 在项目目录中，右键git bash ；执行git init 将当前的目录转化为git仓库
  • 从其他服务器克隆一个已存在的git仓库

工作区文件的四种状态

  • 未被git管理1.未跟踪 untracked
  • 2.未修改 unmodified
  • 3.已修改 modified
  • 4.已暂存 staged

撤销对文件的修改

git reset HEAD 要移除的文件的名称 git reset HEAD . 移除所有暂存区文把工作区中对应文件的修改还原成Git仓库中所保存的版本
操作的结果：所有的修改会丢失且无法恢复

跳过使用暂存区

git标准的工作流程是工作区-暂存区-git仓库，为了简化流程可以简化为工作区-git仓库 使用git commit -a -m '描述信息'会把所有已经跟踪过的文件暂存起来一并提交，跳过git add步骤

移除文件

两种移除方式，一是移除git仓库和工作区中的文件git rm -f 指定文件名; 二是移除git仓库中的文件，保留工作区中的文件git rm --cached 指定文件名

忽略文件

有一些不需要纳入git管理也不希望出现在为跟踪文件列表，这种情况我们可以创建一个名为.gitignore的配置文件，列出要忽略的文件匹配模式，该配置文件以#开头为注释，以/结尾的是目录，以/开头的是放置递归，以！开头表示取反，可以使用glob模式进行文件和文件夹的匹配（glob指简化了的正则表达式）

glob模式 *表示匹配零个或多个字符 [abc]匹配任何一个列在[]中的字符（这个案例里匹配一个a或一个b或一个c） ?只匹配一个任意字符 [0-9]，表示匹配所有0-9的数字 **表示匹配任意中间目录，比如a/**/z可以匹配a/b/z或者 a/l/z

查看提交历史

git log

  • git log 按时间先后顺序列出所有的提交历史，最近的提交排在最上面
  • git log -n 只展示最新的n条提交历史，数字按需填写
  • git log -2 --pretty=oneline 在一行上展示最近两条提交历史（数字可以省略）
  • git log -2 --pretty=format:"%h|%an|%ar|%s" 一行上展示最近两条提交历史的信息，自定义输出的格式；依次是提交的简写哈希值，作者名字，作者修订日期，提交说明；中间的|是自定义的分隔符，打印信息可以选择其中几项
  • q退出查看

回退到指定版本

  1. 一行打印出所有的提交历史 git log --pretty=oneline
  2. 根据提交历史标识符进行版本切换 git reset --hard 标识符
  3. 回到旧版本后，使用git reflog --pretty=oneline 查看命令操作的历史（git log已经不能查看历史操作）
  4. 再次 git reset --hard 标识符 回到响应版本

git终端命令

  • git config --global user.name "liud" 全局配置用户名，只需要配置一次即可
  • git config --global user.email "2510736908@qq.com" 全局配置邮箱名
  • git init 初始化git库
  • git config --list --global --list罗列--global全局配置 的信息
  • git config user.name/email 查看指定的全局配置项
  • git help config 获取帮助信息
  • git config -h 在终端获取帮助信息
  • get status 查看文件处于什么状态 精简方式 get status --short/-s 未跟踪文件前边有??标记 绿色A表示新添加到暂存区的文件 已修改未被提交到暂存区：红色M 已修改被提交到暂存区：绿色M
  • git add 开始跟踪一个文件 跟踪的文件会显示A/changes to be committed
  • git add . 把所有修改和新增过的文件加入暂存区
  • git reset HEAD 要移除的文件的名称
  • git reset HEAD . 移除所有暂存区文件
  • git commit -m '可以对提交的内容做进一步的描述' 提交暂存区文件，-m提交到master
  • git checkout -- jjj.html (以jjj.html文件为例)
  • git rm -f 指定文件名 移除git仓库和工作区中的文件
  • git rm --cached 指定文件名 移除git仓库中的文件，保留工作区中的文件
  • git log 查看提交历史
  • clear 清空终端内容（快捷键ctrl+l）
